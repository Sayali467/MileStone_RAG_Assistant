1. OVERVIEW
-----------
This project involves the design and development of a lightweight FAQ assistant
that provides factual, citation-backed answers about mutual fund schemes in India.
The system must operate exclusively using publicly available, official sources such
as AMC websites, SEBI disclosures, and AMFI portals. It must not generate, infer,
or compute any investment advice, return projections, or portfolio recommendations.

Every response generated by the assistant must include exactly one verifiable source
URL, and answers must remain strictly within the bounds of factual scheme data.

--------------------------------------------------------------------------------

2. PROBLEM CONTEXT
------------------
Retail investors and support/content teams frequently encounter repetitive,
factual queries about mutual fund scheme parameters — such as expense ratios,
exit loads, lock-in periods, SIP minimums, risk ratings, and how to retrieve
account or tax-related statements. Currently, these queries require manual lookup
across multiple AMC and regulatory pages, resulting in inefficiency and the risk
of misinformation from unofficial sources.

Target Users:
  - Retail users comparing mutual fund schemes before investing.
  - Customer support teams handling high-volume, repetitive MF FAQs.
  - Content teams requiring accurate, source-linked scheme information.

--------------------------------------------------------------------------------

3. SCOPE
--------
3.1 AMC & Scheme Selection
    - Select ONE Asset Management Company (AMC).
    - Choose 3–5 schemes under that AMC, ensuring coverage across at least:
        * One large-cap or index fund
        * One flexi-cap fund
        * One ELSS (Equity Linked Savings Scheme)
        * One Debt 
        *One Hybrid


3.2 Corpus Collection
    - Collect 15–25 publicly accessible pages from official sources:
        * AMC website: factsheets, Key Information Memorandum (KIM), Scheme
          Information Document (SID), fee/charges pages.
        * SEBI: scheme-level regulatory filings or disclosures.
        * AMFI: riskometer classifications, benchmark data, scheme FAQs.
        * Official statement/tax-document download guides.
    - Source restrictions:
        * Public sources ONLY — no third-party blogs, aggregators, or paywalled
          content.
        * No screenshots of app back-end interfaces.

--------------------------------------------------------------------------------

4. SYSTEM REQUIREMENTS
-----------------------
4.1 Functional Requirements
    FR-01: The assistant shall answer factual queries only.
           Supported query categories include, but are not limited to:
             - Expense ratio of a scheme
             - Exit load structure and applicability
             - Minimum SIP / lump-sum investment amount
             - ELSS lock-in period
             - Riskometer category and benchmark index
             - How to download a capital gains or account statement

    FR-02: Every answer must include exactly one citation link pointing to an
           official source from the collected corpus.

    FR-03: The assistant must detect and refuse opinionated or advisory queries
           (e.g., "Should I buy this fund?", "Which fund should I invest in?")
           with a polite, pre-defined refusal message and a link to a relevant
           educational resource (e.g., AMFI investor education page).

    FR-04: The UI must display:
             - A welcome message
             - 3 pre-loaded example questions to guide first-time users
             - A persistent disclaimer: "Facts only. No investment advice."

    FR-05: Each answer must be ≤ 3 sentences in length.

    FR-06: Each answer must include the label: "Last updated from sources: <date>"
           to ensure transparency about data freshness.

4.2 Non-Functional Requirements
    NFR-01: NO Personally Identifiable Information (PII) shall be accepted,
            processed, or stored. Prohibited fields include:
              - PAN / Aadhaar numbers
              - Bank account numbers
              - OTPs, email addresses, or phone numbers
            The system must not prompt for, log, or retain any such data.

    NFR-02: NO performance claims shall be generated. The assistant must not
            compute, compare, or project returns. If a return-related query is
            received, the assistant must redirect the user to the official
            factsheet link.

    NFR-03: The system must be lightweight and suitable for deployment as a
            hosted web app like vercel, or Jupyter/Colab notebook.

5. TECHNICAL CONSTRAINTS
-------------------------
  - All document sources must be publicly accessible URLs (no login required).
  - The assistant must NOT use third-party financial blogs, social media, or
    unverified sources as part of its knowledge corpus.
  - The system must include guardrails to prevent hallucination on unsupported
    queries.
  - No back-end screenshots or proprietary data may be included in the corpus.
  Data and Sources
Use only official public sources (AMC, AMFI, SEBI)
Do not use third-party blogs or aggregator websites
Privacy and Security
Do not collect, store, or process:
PAN or Aadhaar numbers
Account numbers
OTPs
Email addresses or phone numbers
Content Restrictions
No investment advice or recommendations
No performance comparisons or return calculations
For performance-related queries, provide a link to the official factsheet only
Transparency
Responses must be short, factual, and verifiable
Every answer must include a source link and last updated date


 Refusal Handling
The assistant must refuse non-factual or advisory queries, such as:
“Should I invest in this fund?”
“Which fund is better?”
Refusal responses should:
Be polite and clearly worded
Reinforce the facts-only limitation
Provide a relevant educational link (e.g., AMFI or SEBI resource)

6. DELIVERABLES
---------------
The following artifacts must be submitted upon project completion:

  D-01: WORKING PROTOTYPE
        A functional chatbot accessible via:
          (a) A hosted web app link (preferred), OR
          (b) A ≤ 3-minute demo video if live hosting is not feasible.

  D-02: SOURCE LIST
        A structured file (CSV or Markdown) listing all 15–25 URLs used in
        the corpus, including the AMC/SEBI/AMFI page name and retrieval date.

  D-03: README
        A documentation file covering:
          - Setup and installation steps
          - Scope declaration (AMC name + scheme names)
          - Known limitations of the assistant

  D-04: SAMPLE Q&A FILE
        A document containing 5–10 representative queries with the assistant's
        actual answers and corresponding source links.

  D-05: DISCLAIMER SNIPPET
        The exact disclaimer text displayed in the UI, confirming:
          - Facts-only scope
          - No investment advice provided
          - Official source citations only


Expected Deliverables
README Document
Setup instructions
Selected AMC and schemes
Architecture overview (RAG approach)
Known limitations
Disclaimer Snippet
“Facts-only. No investment advice.”

Success Criteria
Accurate retrieval of factual mutual fund information
Strict adherence to facts-only responses
Consistent inclusion of valid source citations
Proper refusal of advisory queries
Clean, minimal, and user-friendly interface

Summary
The goal is to build a trustworthy, transparent, and compliant mutual fund FAQ assistant that prioritizes accuracy over intelligence. The system should ensure that users receive only verified, source-backed financial information, without any advisory bias or speculative content.


--------------------------------------------------------------------------------

7. OUT OF SCOPE
---------------
The following are explicitly excluded from this project:
  - Portfolio recommendations or buy/sell signals
  - Return computations or fund performance comparisons
  - User authentication or account-linked features
  - Real-time data feeds or live NAV integration
  - Collection or storage of any user personal data
